# Research Report Rubric (Quality Bar)

## What “good” looks like
- Claims are tied to sources.
- The report distinguishes: established evidence vs. conjecture.
- It includes enough procedural detail that a reader could replicate the literature review.
- It surfaces uncertainty and gaps explicitly.

## Citation Discipline
When stating a factual claim (dataset size, metric, result), cite a source nearby.
Prefer citations to primary sources rather than secondary summaries.
Never invent URLs, DOIs, repository names, PDF links, dataset locations, or citation metadata.
If a source URL was not present in retrieved evidence, write the citation metadata you know and mark the URL as missing instead of fabricating a link.
Use the exact retrieved URL for every markdown link.

## What to avoid
- Vague “recent works show…” statements without citations.
- Over-optimistic SOTA claims without comparable evaluation settings.
- Copying abstracts verbatim instead of synthesizing.
- Broken or guessed links such as publisher homepages when the actual paper URL was not retrieved.

## Deliverables for the Workspace
- Save the final report to `bib_pdf/` as markdown.
- Save any acquired PDFs to `bib_pdf/`.
- When possible, highlight which PDFs are the most foundational (top 3–7).
